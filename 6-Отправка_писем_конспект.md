## Введение: как письмо попадает к получателю

```
┌─────────────┐    mail() / SMTP     ┌──────────────┐    SMTP/DNS    ┌─────────────┐
│ PHP-приложение│ ─────────────────► │ MTA (sendmail,│ ─────────────► │ Почтовый    │
│             │                      │ Postfix, MS)  │                │ сервер      │
└─────────────┘                      └──────────────┘                │ получателя  │
                                                                      └─────────────┘
```

| Этап | Что происходит |
|------|----------------|
| **1. Формирование** | PHP собирает заголовки и тело (текст, HTML, файлы) |
| **2. Передача MTA** | `mail()` отдаёт письмо локальному агенту или SMTP-клиент подключается к серверу |
| **3. Доставка** | Почтовые серверы по DNS (MX-записи) доставляют письмо в ящик |

**MTA** (Mail Transfer Agent) — программа, которая **пересылает** почту (sendmail, Postfix, Exim, Microsoft Exchange).

## 1. Функция mail()

### Что это

**`mail()`** — встроенная функция PHP для отправки email через **локальный почтовый агент** сервера (sendmail и аналоги). PHP **не отправляет** письмо напрямую в интернет — она передаёт его ОС/MTA.

```php
mail(
    string $to,
    string $subject,
    string $message,
    array|string $additional_headers = [],
    string $additional_params = ''
): bool
```

| Параметр | Описание |
|----------|----------|
| `$to` | Получатель(и): `user@example.com` или `Name <user@example.com>` |
| `$subject` | Тема (кодировка — см. ниже) |
| `$message` | Тело письма |
| `$additional_headers` | Заголовки: From, Reply-To, MIME, Content-Type |
| `$additional_params` | Доп. параметры для sendmail (редко) |

### Минимальный пример

```php
<?php
$to = 'client@example.com';
$subject = 'Заказ принят';
$message = "Здравствуйте!\n\nВаш заказ №123 оформлен.\n";
$headers = [
    'From: Shop <noreply@shop.ru>',
    'Reply-To: support@shop.ru',
    'Content-Type: text/plain; charset=UTF-8',
    'X-Mailer: PHP/' . phpversion(),
];

$success = mail($to, $subject, $message, implode("\r\n", $headers));

if (!$success) {
    error_log('mail() failed for order #123');
}
```

> Строки заголовков разделяются **`CRLF`** (`\r\n`), не только `\n`.

### HTML-письмо через mail()

```php
<?php
$to = 'client@example.com';
$subject = '=?UTF-8?B?' . base64_encode('Ваш заказ') . '?=';

$message = <<<'HTML'
<!DOCTYPE html>
<html>
<body>
  <h1>Спасибо за заказ!</h1>
  <p>Номер заказа: <strong>123</strong></p>
</body>
</html>
HTML;

$headers = implode("\r\n", [
    'MIME-Version: 1.0',
    'From: Shop <noreply@shop.ru>',
    'Content-Type: text/html; charset=UTF-8',
    'Content-Transfer-Encoding: 8bit',
]);

mail($to, $subject, $message, $headers);
```

### Кодировка темы (кириллица)

Тема с не-ASCII символами кодируется по **RFC 2047**:

```php
function encodeSubject(string $subject): string
{
    return '=?UTF-8?B?' . base64_encode($subject) . '?=';
}

mail($to, encodeSubject('Подтверждение регистрации'), $body, $headers);
```

### Настройка в php.ini

```ini
[mail function]
SMTP = localhost
smtp_port = 25
sendmail_from = noreply@shop.ru
sendmail_path = "/usr/sbin/sendmail -t -i"
```

| ОС | Поведение |
|----|-----------|
| **Linux** | Обычно `sendmail_path` → Postfix/sendmail |
| **Windows** | `SMTP` и `smtp_port` — прямое подключение к SMTP (ограниченно) |

### Плюсы и минусы mail()

| Плюсы | Минусы |
|-------|--------|
| Встроена в PHP, не нужны библиотеки | Зависимость от настройки сервера |
| Простой текстовый сценарий | Плохо подходит для HTML + вложений |
| | Письма часто попадают в **спам** |
| | Нет аутентификации, очередей, retry |
| | Сложно тестировать локально |
| | `bool` — мало информации об ошибке |

> **На экзамене:** `mail()` — **низкоуровневый** способ; в production чаще используют **SMTP + библиотеки** (PHPMailer, Symfony Mailer).

### Безопасность

```php
// ❌ Опасно: подстановка из $_POST без фильтрации
mail($_POST['email'], 'Hi', '...', 'From: ' . $_POST['from']);

// ✅ Валидация, фиксированный From
$to = filter_var($_POST['email'], FILTER_VALIDATE_EMAIL);
if ($to === false) {
    throw new InvalidArgumentException('Invalid email');
}
```

**Header injection:** если в `$to` или заголовках попадут `\r\n`, злоумышленник может добавить свои заголовки. Использовать `filter_var` и не подставлять сырой ввод в заголовки.

## 2. Отправка через SMTP

### Что такое SMTP

**SMTP** (Simple Mail Transfer Protocol) — протокол **передачи** почты между серверами. Клиент (PHP-приложение) подключается к SMTP-серверу, проходит аутентификацию и передаёт письмо.

```
PHP (PHPMailer / Symfony Mailer)
        │
        │ TCP 587 (STARTTLS) или 465 (SSL)
        ▼
┌───────────────────┐
│ smtp.yandex.ru    │  ← почтовый провайдер / свой Postfix
│ smtp.gmail.com    │
│ mailgun, SendGrid │
└───────────────────┘
        │
        ▼
   Получатель @mail.ru
```

### Типичные порты

| Порт | Режим | Описание |
|------|-------|----------|
| **25** | Plain / STARTTLS | Классический MTA-to-MTA (часто блокируется хостингом) |
| **587** | **STARTTLS** | Рекомендуемый для клиентов (submission) |
| **465** | **SSL/TLS** | Устаревший «SMTPS», всё ещё используется |
| **2525** | Альтернатива | Некоторые SaaS (Mailgun и др.) |

### Зачем SMTP вместо mail()

| Возможность | mail() | SMTP |
|-------------|--------|------|
| Логин и пароль | Нет | **Да** |
| Внешний сервис (Gmail, Yandex) | Сложно | **Да** |
| TLS/SSL | Зависит от MTA | **Явно** |
| Диагностика ошибок | Слабая | **Детальные коды** |
| Очередь, retry | Нет | Через библиотеку/сервис |
| Доставляемость (SPF/DKIM) | Сложнее | Настраивается на домене |

### PHPMailer + SMTP (популярный пример)

```php
<?php
use PHPMailer\PHPMailer\PHPMailer;
use PHPMailer\PHPMailer\Exception;

require 'vendor/autoload.php';

$mail = new PHPMailer(true);

try {
    $mail->isSMTP();
    $mail->Host       = 'smtp.yandex.ru';
    $mail->SMTPAuth   = true;
    $mail->Username   = 'noreply@shop.ru';
    $mail->Password   = getenv('SMTP_PASSWORD'); // не хардкодить!
    $mail->SMTPSecure = PHPMailer::ENCRYPTION_STARTTLS;
    $mail->Port       = 587;
    $mail->CharSet    = 'UTF-8';

    $mail->setFrom('noreply@shop.ru', 'Магазин');
    $mail->addAddress('client@example.com', 'Клиент');
    $mail->addReplyTo('support@shop.ru', 'Поддержка');

    $mail->isHTML(true);
    $mail->Subject = 'Заказ принят';
    $mail->Body    = '<h1>Спасибо!</h1><p>Заказ №123</p>';
    $mail->AltBody = 'Спасибо! Заказ №123';

    $mail->send();
} catch (Exception $e) {
    error_log('Mailer Error: ' . $mail->ErrorInfo);
}
```

### Symfony Mailer (современный подход)

```php
<?php
use Symfony\Component\Mailer\Transport;
use Symfony\Component\Mailer\Mailer;
use Symfony\Component\Mime\Email;

$dsn = 'smtp://user:pass@smtp.example.com:587';
$transport = Transport::fromDsn($dsn);
$mailer = new Mailer($transport);

$email = (new Email())
    ->from('noreply@shop.ru')
    ->to('client@example.com')
    ->subject('Заказ принят')
    ->text('Заказ №123 оформлен.')
    ->html('<p>Заказ <strong>№123</strong> оформлен.</p>');

$mailer->send($email);
```

**DSN-формат:**

```
smtp://USER:PASS@HOST:PORT?encryption=tls&auth_mode=login
smtps://USER:PASS@HOST:465
```

### Laravel Mail (обёртка)

```php
// .env
// MAIL_MAILER=smtp
// MAIL_HOST=smtp.mailtrap.io
// MAIL_PORT=2525
// MAIL_USERNAME=...
// MAIL_PASSWORD=...

Mail::to($user)->send(new OrderPlaced($order));
```

```php
// app/Mail/OrderPlaced.php
class OrderPlaced extends Mailable
{
    public function __construct(public Order $order) {}

    public function build(): self
    {
        return $this->subject('Заказ принят')
            ->view('emails.order-placed')
            ->with(['order' => $this->order]);
    }
}
```

### SaaS-сервисы (SMTP API)

Вместо своего Postfix часто используют:

| Сервис | Назначение |
|--------|------------|
| **Mailtrap** | Тестовый inbox (dev/staging) |
| **SendGrid, Mailgun, Amazon SES** | Массовая транзакционная почта |
| **Postmark** | Транзакционные письма |

Они дают SMTP-учётные данные или HTTP API; PHP подключается через SMTP DSN или SDK.

### Аутентификация и шифрование

| Механизм | Суть |
|----------|------|
| **SMTPAuth** | Логин + пароль (или app-password) |
| **STARTTLS** | Сначала plain, затем upgrade до TLS (порт 587) |
| **SSL/TLS** | Шифрование с первого байта (порт 465) |

### Доставляемость (кратко для экзамена)

Чтобы письма не шли в спам, настраивают DNS записи домена:

| Запись | Назначение |
|--------|------------|
| **SPF** | Какие серверы могут отправлять от имени домена |
| **DKIM** | Криптоподпись письма |
| **DMARC** | Политика при несовпадении SPF/DKIM |

From должен быть с **вашего домена**, с которого реально отправляет SMTP.

### Сравнение: mail() vs SMTP

```
mail()                          SMTP + PHPMailer
──────                          ─────────────────
PHP → sendmail (локально)       PHP → smtp.server.com:587
Без логина                      LOGIN + PASSWORD
Настройка php.ini               .env / config
Спам-риск выше                  Контроль выше
```

## 3. Особенности отправки писем с файлами

### Почему это сложнее простого текста

Письмо с вложением — это **multipart**-сообщение: несколько частей (текст, HTML, файлы) в одном письме по стандарту **MIME** (Multipurpose Internet Mail Extensions).

```
Письмо (multipart/mixed)
├── Часть 1: text/plain  (альтернативный текст)
├── Часть 2: text/html   (HTML-версия)
└── Часть 3: application/pdf (вложение invoice.pdf)
```

Без правильной структуры почтовые клиенты покажут **сырой base64** вместо файла.

---

### 3.1 Основные MIME-типы

| Content-Type | Пример |
|--------------|--------|
| `text/plain` | Обычный текст |
| `text/html` | HTML-письмо |
| `multipart/alternative` | plain + html (клиент выберет лучшее) |
| `multipart/mixed` | Тело + вложения |
| `application/pdf` | PDF |
| `image/png` | Картинка |
| `application/octet-stream` | Любой бинарный файл |

### 3.2 Кодирование вложений

Бинарные данные в email передаются в **Base64** (или quoted-printable для текста).

```
Content-Transfer-Encoding: base64

JVBERi0xLjQKJeLjz9MKMy...
```

| Кодировка | Когда |
|-----------|-------|
| **base64** | Файлы, изображения |
| **quoted-printable** | Текст с редкими не-ASCII |
| **8bit** | UTF-8 текст без бинарника |

---

### 3.3 Вложение через PHPMailer

```php
<?php
$mail = new PHPMailer(true);
// ... настройка SMTP ...

$mail->setFrom('noreply@shop.ru', 'Магазин');
$mail->addAddress('client@example.com');
$mail->Subject = 'Счёт по заказу №123';

$mail->Body = '<p>Во вложении счёт в формате PDF.</p>';
$mail->isHTML(true);

// Файл с диска
$mail->addAttachment(
    '/var/app/storage/invoices/invoice-123.pdf',
    'invoice-123.pdf',           // имя для получателя
    'base64',
    'application/pdf'
);

// Второе вложение
$mail->addAttachment('/path/to/terms.docx', 'Условия.docx');

$mail->send();
```

| Метод | Назначение |
|-------|------------|
| `addAttachment($path, $name)` | Файл с диска |
| `addStringAttachment($data, $name)` | Файл из строки/памяти (без записи на диск) |
| `addEmbeddedImage($path, $cid)` | Картинка **внутри** HTML (`cid:`) |

---

### 3.4 Вложение через Symfony Mime

```php
<?php
use Symfony\Component\Mime\Email;
use Symfony\Component\Mime\Part\DataPart;

$email = (new Email())
    ->from('noreply@shop.ru')
    ->to('client@example.com')
    ->subject('Счёт №123')
    ->html('<p>Счёт во вложении.</p>')
    ->attachFromPath('/path/to/invoice-123.pdf', 'invoice.pdf', 'application/pdf');

// Из содержимого в памяти
$email->attach($pdfBinary, 'invoice.pdf', 'application/pdf');

$mailer->send($email);
```

### 3.5 Inline-изображения в HTML (не путать с вложением)

```php
// PHPMailer
$mail->AddEmbeddedImage('/path/logo.png', 'logo_cid', 'logo.png');
$mail->Body = '<img src="cid:logo_cid" alt="Logo">';
```

Получатель видит картинку **в теле** письма; в списке вложений её может не быть.

### 3.6 Ручная сборка multipart (mail()) — для понимания

На экзамене могут спросить **структуру**, даже если на практике используют библиотеку:

```php
<?php
$boundary = '=_Part_' . md5(uniqid((string) time(), true));
$to = 'client@example.com';
$subject = encodeSubject('Документы по заказу');

$body = "--{$boundary}\r\n";
$body .= "Content-Type: text/plain; charset=UTF-8\r\n\r\n";
$body .= "См. вложение.\r\n\r\n";

$filePath = '/path/report.pdf';
$fileContent = chunk_split(base64_encode(file_get_contents($filePath)));

$body .= "--{$boundary}\r\n";
$body .= "Content-Type: application/pdf; name=\"report.pdf\"\r\n";
$body .= "Content-Transfer-Encoding: base64\r\n";
$body .= "Content-Disposition: attachment; filename=\"report.pdf\"\r\n\r\n";
$body .= $fileContent . "\r\n";
$body .= "--{$boundary}--\r\n";

$headers = implode("\r\n", [
    'MIME-Version: 1.0',
    'From: Shop <noreply@shop.ru>',
    "Content-Type: multipart/mixed; boundary=\"{$boundary}\"",
]);

mail($to, $subject, $body, $headers);
```

| Заголовок вложения | Смысл |
|--------------------|-------|
| `Content-Disposition: attachment` | Скачиваемый файл |
| `Content-Disposition: inline` | Показать внутри письма |
| `filename=` | Имя файла для пользователя |

### 3.7 Ограничения и особенности

| Ограничение | Типичное значение |
|-------------|-------------------|
| Размер письма | 10–25 МБ (зависит от SMTP/провайдера) |
| Количество вложений | Разумно ≤ 5–10 |
| Имена файлов | Только безопасные символы; кириллица — RFC 2231 / encoded |

### Безопасность при работе с файлами

```php
// ❌ Опасно: прикрепить любой путь от пользователя
$mail->addAttachment($_GET['file']);

// ✅ Проверка: файл из разрешённой директории, whitelist расширений
$allowed = ['pdf', 'docx'];
$path = realpath($storageDir . '/' . basename($filename));
if ($path === false || !str_starts_with($path, $storageDir)) {
    throw new RuntimeException('Invalid file');
}
```

| Риск | Мера |
|------|------|
| **Path traversal** | `basename()`, проверка realpath |
| **Вирусы** | Антивирус на upload, не исполнять вложения |
| **Утечка данных** | Не прикреплять лишние файлы, логировать факт отправки |
| **Память** | Большие файлы — stream, не `file_get_contents` на 100 МБ |

### Генерация PDF на лету (без сохранения на диск)

```php
$pdfContent = $pdfGenerator->render($order); // бинарная строка

$mail->addStringAttachment(
    $pdfContent,
    'order-123.pdf',
    'base64',
    'application/pdf'
);
```

### 3.8 HTML + текст + вложение (полная структура)

```
multipart/mixed
└── multipart/alternative
    ├── text/plain
    └── text/html
└── application/pdf (attachment)
```

PHPMailer и Symfony Mailer собирают это **автоматически** при `isHTML(true)` + `addAttachment()`.

## Шпаргалка перед экзаменом

| Вопрос | Краткий ответ |
|--------|---------------|
| **mail()** | Встроенная функция → локальный sendmail/MTA |
| **SMTP** | Протокол; PHP через PHPMailer/Symfony с логином и TLS |
| **Порт 587** | STARTTLS (рекомендуется) |
| **Вложения** | MIME multipart + base64; лучше библиотека |
| **inline vs attachment** | inline — в HTML (`cid:`); attachment — файл |
| **Почему не mail() на проде** | Спам, нет auth, сложный MIME |

### Сравнительная таблица

| | mail() | SMTP + библиотека |
|---|--------|-------------------|
| Настройка | php.ini | .env, DSN |
| HTML | Вручную | `isHTML()`, `->html()` |
| Файлы | Вручную multipart | `addAttachment()` |
| Ошибки | `true/false` | Exception, ErrorInfo |
| Production | Редко | **Стандарт** |

### Типичные ошибки

| Ошибка | Правильно |
|--------|-----------|
| `\n` вместо `\r\n` в заголовках | `\r\n` |
| Тема кириллицей без кодировки | `=?UTF-8?B?...?=` |
| Пароль SMTP в git | `.env`, secrets |
| Вложение без boundary | multipart + библиотека |
| From с чужого домена | SPF/DKIM своего домена |
